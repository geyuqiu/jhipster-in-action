JHipster Domain Language (JDL)

                    
                        entity Project {
                            name String required
                        }
                        // create entities: Company & Standard in the same way...

                        relationship ManyToMany {
                            Project{company(name)} to Company{project(name)}
                        }

                        relationship OneToMany {
                            Company{standard} to Standard{company(name) required}
                        }


                        dto * with mapstruct
  • define entity relationships in JDL
  • call import jdl, and boom, the defined entities are generated in Spring Boot and Angular

Testing

  • JUnit5
  • Jest (Jasmin)
  • Protractor
  • e2e-test for login
  • password for the default users like test or system are stored in encrypted form in db, but was used as plain text in tests
  • change the password / delete test user, have to make adjustment in tests
  • Protractor e2e tests for updating password
  • except some functionality like jasmine.createSpyObj, jest provides most of the functions of jasmin
  • .overrideTemplate(CompanyComponent, '') has to be disabled if you want add unit tests for interaction with UI-Elements
  • angular.json: skipTests false -> true
  • not all angular cli commands are supported
  • ng-mock
  • Import/Export e.g. FormsModule, MaterialModule in TestModule, import in Component tests
  • liquibase and faker generate fake data declared in csv files

Where JHipster shines

  • simplification, productivity, consistency
  • for rapid prototyping
  • production-ready
  • RESTful
  • latest frameworks
  • differentiates from other (yeoman) generators:
    • fullStack
    • foundation for TDD: Unit-, Integration- and UI-Tests
    • popularity
  • consistency of created components e.g.

Why using JHipster

  • little budget
  • working prototype after first product increment to get first feedback from stakeholder
  • Full stack developer -> rapid prototyping
  • Spring Boot developer -> learn SPA
  • Angular Developer -> React / Vue

but ... there were challenges

  • experience with Spring Boot
  • and at least 1 SPA-Framework
  • crud web pages: not responsive on mobile clients
  • Understanding:
    • jdl
    • JHipster way of writing code

Unclear

  • too much features
  • utilize JHipster in an ongoing project with its blueprints
  • demarcation between JHipster specific and e.g. Angular / Spring Boot issues
